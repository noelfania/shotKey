# 코딩·주석 규칙

## 코드 스타일

- 변수명과 식별자는 관례적이고 읽기 쉬운 이름을 사용한다.
- 불필요한 추상화보다 현재 구조에서 바로 읽히는 상태 이름과 함수 이름을 우선한다.
- 주석이 꼭 필요하다면 짧고 명확한 **한글**로 작성한다.

## JSDoc 규칙

함수 선언부 바로 위에 완전한 JSDoc 블럭을 둡니다.

- 첫 줄: 기능 요약
- 모든 인수: `@param`
- 반환값이 있으면: `@returns`
- 의미 있는 부수효과가 있으면 본문 문장으로 간단히 명시

예외:

- 인수가 없는 함수는 불필요한 `@param`을 만들지 않습니다.
- 반환값이 없는 함수/콜백은 억지로 `@returns`를 달지 않습니다.

## 인라인 주석

다음 구간에만 짧은 한글 라인 주석을 둡니다.

- 복잡한 계산식
- 큐 삽입 규칙
- 타이머/이벤트 처리
- 상태 전이 구간

넣지 않는 것:

- 자명한 대입문
- JSX 마크업 구조 설명

## CSS 주석

- 함수형 블럭 주석 대신 **구간 시작점 섹션 헤더** 주석
- 애니메이션, 하이라이트, 반응형처럼 의도가 중요한 규칙에만 포인트 주석
- CSS 속성 하나하나를 번역하는 식의 설명 주석은 넣지 않습니다.
