# Gameplay Review

## 목적

이 스킬은 타이핑 플레이 경험과 UI 변경이 기존 프로젝트 규칙과 충돌하지 않는지 빠르게 점검하기 위한 프로젝트용 샘플 템플릿이다.

## 기본 참조 문서

- `.cursor/rules/project-standards.mdc`
- `.cursor/rules/project-design-spec.md`

## 주로 쓰는 상황

- `src/App.tsx`에서 게임 상태, 판정, 키 입력 처리 로직을 수정할 때
- 현재 문자 카드, 다음 문자 미리보기, HUD, `keyboard-panel` 표시를 바꿀 때
- `전체`, `왼손`, `오른손`, `무한모드` 동작을 건드릴 때

## 작업 순서

1. 먼저 변경 범위를 요약한다.
2. 변경이 아래 핵심 규칙과 맞는지 확인한다.
3. UI 텍스트와 실제 동작이 어긋나지 않는지 본다.
4. 빠진 테스트 또는 수동 확인 항목이 있으면 적는다.

## 핵심 체크리스트

- 현재 입력 문자가 1개 기준으로 명확히 보이는가
- 다음 문자 미리보기가 깨지지 않는가
- 정답 시 즉시 다음 문자로 진행하는 흐름이 유지되는가
- 오답 시 `MISS`와 입력 잠금 규칙이 깨지지 않는가
- 집중 게이지 시작, 감소, 종료 규칙이 유지되는가
- `전체`, `왼손`, `오른손`, `무한모드` 설명과 실제 동작이 맞는가
- 키보드 패널 강조와 실제 입력 대상이 어긋나지 않는가

## 출력 형식

아래 형식으로 답한다.

```markdown
## 변경 요약
- ...

## 규칙 일치 여부
- 일치: ...
- 불일치: ...

## 확인 필요 항목
- ...
```

## 주의

- 새 규칙을 만들기보다 기존 `.cursor/rules` 문서를 우선 참조한다.
- 문서에 없는 새 정책을 임의로 확정하지 않는다.
- 애매하면 설계서보다 현재 프로젝트 규칙 문서를 우선 기준으로 삼는다.
